# apps/construction/forms.py — file validation
def _validate_attachment(file_obj):
    max_size = 5 * 1024 * 1024
    if file_obj.size > max_size:
        raise ValidationError("File must be 5MB or smaller.")
    allowed = {".pdf", ".jpg", ".jpeg", ".png"}
    if not any(file_obj.name.lower().endswith(ext) for ext in allowed):
        raise ValidationError("Only PDF, JPG, or PNG files are accepted.")

Key Decisions & Trade-offs

§7
Option A (lean core, 4 models) over Option B (7 models) Honoured ADR-002 — no abandoned-shell anti-pattern. Property, Product, Equipment deferred until real content exists; each is a ~1-day follow-up PR when triggered by a client need.
3 separate optional FileField inputs instead of a custom MultipleFileField widget Django 5's multiple=True on FileField still requires custom widget gymnastics. Three explicit slots is pragmatic, validates per-file at form level, and produces a flat QuoteAttachment table that's easy to query.
Construction-specific navbar with hub backlink Refactored core/base.html to wrap navbar in {% block navbar %}. base_construction.html overrides with a dedicated nav. Adds a small "← Bejundas Group" backlink so users don't feel trapped in the vertical.
Email as EmailMultiAlternatives (plain + HTML) Recipients on text-only mail clients still get the data; everyone else gets the branded notification with "Review in Admin" CTA. Templates split into .txt and .html partials in templates/emails/.
Idempotent RunPython seed migration with update_or_create Re-running the migration won't duplicate rows. Reverse code removes only the seeded entries by slug/name. Safe to re-run on staging/dev DBs that already have the seed.
Tests use "TestPrefix" names; sector filter test wipes Project table first Seed migration loads 10 projects into every test DB. Model tests now use unique prefixed names; the sector-filter test deletes all projects in setup_method so counts are deterministic.
