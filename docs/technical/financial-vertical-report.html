# apps/financial/forms.py — shared phone validator
def _validate_phone(value: str) -> str:
    cleaned = (value or "").strip()
    digits = "".join(ch for ch in cleaned if ch.isdigit())
    if len(digits) < 9:
        raise ValidationError("Enter a valid phone number (minimum 9 digits).")
    return cleaned

Key Decisions & Trade-offs

§7
Two lead models (LoanInquiry + InvestmentInquiry), not one discriminated model The funnels are fundamentally different — loan: applied→reviewed→approved→disbursed; investment: inquired→contacted→committed→funded→active→matured. Separate models keep admin pipelines clean and per-funnel bulk actions readable. Worth the slight code duplication.
Offering FK on InvestmentInquiry set server-side from ?offering= — never a form field Prospects can't tamper with which round they're inquiring about. The view's _resolve_offering reads the query string, validates the slug against the DB, and attaches the FK on POST. The "Inquire About This Round" CTA on the detail page passes the slug through; the standalone form page accepts an unattached inquiry.
Safe regulatory wording on every page — private placement, not deposit, not insured, indicative not guaranteed Public website is more permanent than WhatsApp Status. Every page footer carries the disclaimer; the Investments and Loans funnels carry funnel-specific variants. If BFS later obtains a CMSA/BoT/TIRA licence, swap to the regulator-badge variant in _disclaimer.html.
Disclaimer extracted into a partial with 5 variants — DRY _disclaimer.html with {% include … with variant="loan" %} replaces 8 hardcoded blocks. Single source of truth for legal wording; copy changes happen in one place. Uses {% comment %} (not {# ... #}) for multi-line comments — the latter is single-line only and silently breaks.
EmailMultiAlternatives (plain + branded HTML) for both lead funnels Plain-text always sends; HTML alternative attached. Templates split into templates/emails/loan_inquiry_notification.{txt,html} + investment_inquiry_notification.{txt,html}. Amount formatting uses django.contrib.humanize intcomma (TZS 5,000,000 not 5000000) — registered in INSTALLED_APPS as part of this work.
Slug-based offering detail URLs (not reference_id with slashes) "BFS/IO/2026/01" has slashes — illegal in a path-based URL. Model save() override slugifies the ref into bfs-io-2026-01. SlugField unique; sitemap and detail view use the slug.
Idempotent RunPython seed with update_or_create + targeted reverse Re-running 0002 won't duplicate; reverse removes only seeded slugs/refs/names. Admin-added rows survive a migrate-zero scenario. Seed runs automatically on every manage.py migrate — no separate seed command on cPanel.
Organization JSON-LD on the home page only FinancialService schema with address, phone, parentOrganization link to bejundas.co.tz. Only on the home page (per ldjson best practice — don't fragment entity definitions across the site).
