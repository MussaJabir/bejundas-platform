Bug Found & Fixed

02
NoReverseMatch — 'djdt' is not a registered namespace on all subdomains URL Conf
Fix

The debug toolbar middleware injects itself into every response site-wide, including coming_soon pages. Its template resolves {% url 'djdt:render_panel' %} — but the djdt namespace was only registered in config/urls.py (hub URL conf), not in apps/leads/urls.py. Added the same if settings.DEBUG: include(debug_toolbar.urls) block to the leads URL conf. No-op in production where DEBUG=False and the package is not installed.
