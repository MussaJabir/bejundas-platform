The Change

§1

Verticals previously routed by hostname via django-hosts are now plain Django URL patterns under the apex domain. The coming_soon view receives vertical as a URL kwarg instead of parsing request.get_host().

# Before — config/hosts.py (deleted)
host(r"financial", "apps.leads.urls", name="financial")
host(r"construction", "apps.leads.urls", name="construction")
# ...4 more

# After — apps/leads/urls.py
path("financial/",    views.coming_soon, {"vertical": "financial"}, name="financial"),
path("construction/", views.coming_soon, {"vertical": "construction"}, name="construction"),
# ...4 more
path("technologies/",
     RedirectView.as_view(url="https://bjptechnologies.co.tz", permanent=True),
     name="technologies"),

Why (ADR-007)

§2
Client direction (2026-05-15) Subdomains imply "separate business" (e.g. AWS vs Amazon). Paths imply "section of the parent." Bejundas is a holdings group — paths match the brand reality.
SEO consolidation One canonical domain accumulates link equity instead of fragmenting it across six hostnames.
Operational simplicity One SSL cert, one DNS record, one cPanel domain. Eliminates the wildcard-subdomain Passenger setup that caused the Phase 4 deploy headache.
Trade-off accepted Old subdomain bookmarks 404 — no 301 redirects retained. Acceptable: subdomains were live for one day; site was never publicly announced.
