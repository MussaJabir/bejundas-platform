What Was Built

§1
scripts/deploy.sh — cPanel virtualenv detection Rewrote _activate_venv() to dynamically find the cPanel virtualenv at ~/virtualenv/{app_name}/{python_version}/bin/activate using find, with fallback to local venv/ for dev. Eliminates the hardcoded path that broke on cPanel shared hosting.
.cpanel.yml — Initial deployment tasks Replaced the placeholder cp -R approach with real setup tasks: activate virtualenv → pip install -r requirements.txt → migrate --noinput → collectstatic --noinput --clear → touch tmp/restart.txt. Triggered by cPanel Git Version Control "Deploy HEAD Commit".
config/__init__.py — PyMySQL compatibility shim Added import pymysql; pymysql.install_as_MySQLdb(). Switched requirements.txt from mysqlclient (requires compiled C headers unavailable on cPanel shared hosting) to PyMySQL (pure Python). Transparent drop-in for Django's MySQL backend.
apps/hub/models.py — Service.url field Added url = models.URLField(max_length=300, blank=True, default="") to the Service model. Allows each service card on home and services pages to link directly to its vertical subdomain.
apps/hub/migrations/0002_service_url.py Schema migration adding the url column to hub_service table.
apps/hub/migrations/0003_seed_services.py Data migration seeding 5 Service rows — Financial, Construction, Energies, Farming, Investments — each with title, summary, icon, subdomain URL, and display order. Uses get_or_create for idempotency. Reversible via unseed_services.
scripts/create_admin.py — Superuser bootstrap Standalone script runnable via cPanel Python App "Execute python script" field. Creates the initial admin superuser non-interactively. Skips creation if user already exists. Prints credentials to output with a change-password reminder.
home.html + services.html — Dynamic service links Updated service card anchor tags to use {% if service.url %}{{ service.url }}{% else %}{% url 'hub:services' %}{% endif %}. Each vertical card now links to its own subdomain when a URL is set.

Webhook endpoint: POST /deploy/webhook/ — HMAC-SHA256 validated against GITHUB_WEBHOOK_SECRET. Deploy runs in a detached subprocess so the HTTP response returns immediately (GitHub's 10s timeout is respected). Log written to /tmp/bjp_deploy.log.

Bugs Found & Fixed During Deploy

§3
Bug 1 public_html/index.php served instead of Django app

Root cause: Apache serves static files before Passenger. The default cPanel index.php in public_html/ satisfied the request before Passenger could handle it.

Fix: Deleted public_html/index.php. Without an index file, Apache falls through to Passenger. Verified .htaccess in public_html/ contained the correct Passenger directives (auto-generated by cPanel Python App setup).

FIXED
Bug 2 Passenger error: recursive load of passenger_wsgi.py

Root cause: When the cPanel Python App was created, it overwrote the project's passenger_wsgi.py with its default template. The default file contains wsgi = load_source('wsgi', 'passenger_wsgi.py') — causing infinite recursion as the file loads itself.

Fix: Restored correct passenger_wsgi.py via File Manager: reads .env via django-environ, sets DJANGO_SETTINGS_MODULE, inserts BASE_DIR to sys.path, and calls get_wsgi_application().

FIXED
Bug 3 django.core.exceptions.ImproperlyConfigured: Set the SECRET_KEY — .env "Invalid line" errors

Root cause: The .env file was created in File Manager with leading spaces before every variable (copy-paste indentation), the ALLOWED_HOSTS value wrapped onto a second line, the file content was duplicated 3×, and GITHUB_WEBHOOK_SECRET was truncated. django-environ rejects any line not starting with the variable name directly.

Fix: Rewrote .env from scratch with no leading spaces. Shortened ALLOWED_HOSTS to .bejundas.co.tz (Django's wildcard subdomain format — matches all 7 hostnames in one short line, preventing editor line-wrap).

FIXED
Bug 4 ModuleNotFoundError: No module named 'mysqlclient'

Root cause: mysqlclient requires compiled C development headers (libmysqlclient-dev) which cPanel shared hosting does not expose. Pip install appeared to succeed but the import failed at runtime.

Fix: Switched to PyMySQL (pure Python, no compilation). Added pymysql.install_as_MySQLdb() in config/__init__.py so Django's MySQL backend finds it transparently. No other code changes required.

FIXED
Bug 5 OperationalError: Unknown column 'hub_service.url'

Root cause: Migration 0002_service_url existed in code but had not been applied to the database. Django tried to SELECT the url column in the HomeView queryset before the column existed.

Fix: Ran manage.py migrate --noinput via cPanel Python App Execute python script field. Column added. Migration 0003_seed_services also ran, inserting the 5 vertical service rows.

FIXED
