Architecture: SiteSettings Proxy Pattern

§2

The single SiteSettings singleton row gets seven different admin views via proxy models. No DB schema changes — proxies share the parent table.

# apps/core/models.py
class SiteSettings(BaseModel):
    # ...30+ fields...
    @classmethod
    def get(cls):
        obj, _ = cls.objects.get_or_create(pk="00000000-...")
        return obj

class IdentitySettings(SiteSettings):
    class Meta: proxy = True

class HeroSettings(SiteSettings):
    class Meta: proxy = True

# ...5 more — AboutSettings, MissionVisionSettings, CTASettings,
# ContactSettings, SocialMediaSettings


# apps/core/admin.py
class _BaseSiteSettingsAdmin(ModelAdmin):
    def has_add_permission(self, request):    return False
    def has_delete_permission(self, request, obj=None): return False

    def changelist_view(self, request, extra_context=None):
        model_name = self.model._meta.model_name
        return HttpResponseRedirect(
            reverse(f"admin:core_{model_name}_change", args=[SiteSettings.get().pk])
        )

@admin.register(HeroSettings)
class HeroSettingsAdmin(_BaseSiteSettingsAdmin):
    fieldsets = (("Home Page — Hero Section", {"fields": (...)}),)

Upstream Bug: Unfold Badge Renders "None"

§3
Bug Sidebar badge bubble renders the literal text "NONE" when the badge callback returns Python None

Root cause: Unfold (v0.93.0) wraps the badge callable with django.utils.functional.lazy(callback)(request). Lazy proxies are always truthy in Python — bool(lazy_proxy) returns True regardless of the wrapped value. So Unfold's template guard {% if item.badge_callback %} never short-circuits, and {{ item.badge_callback }} renders str(None) = "None" (then CSS text-transform:uppercase turns it into "NONE").

# Verified in Django shell:
>>> from django.utils.functional import lazy
>>> ln = lazy(lambda r: None)('req')
>>> bool(ln)   # always True
True
>>> str(ln)
'None'

Fix: Override templates/unfold/helpers/app_list_badge.html. Use {% with bv=item.badge_callback|stringformat:"s" %} to coerce the lazy to a real string, then guard with {% if bv and bv != "None" %}. Cleanly hides the badge when the callback returns falsy values without touching the rest of the badge logic.

Fixed in PR #34

Decisions & Trade-offs

§7
Tailwind blue palette over true Bejundas navy as primary The public site uses navy (#1a1a2e) for chrome and blue (#2563eb) for interactive elements. Using navy as Unfold primary.500 would make every button pure-dark (heavy, hard to read). Using the Tailwind blue palette matches the public site's --blue token used in templates, reports, and CSS — same brand world.
Proxy admins over a single monolithic ModelAdmin Same pattern proven on bjp-technologies-web. Splitting SiteSettings into 7 proxy admins gives 7 focused sidebar entries instead of one giant form. Trade-off: 7 ContentType entries in the DB (cosmetic) and slightly chattier admin permissions to manage. Acceptable.
Dropped date_hierarchy mid-Phase-5 It triggers MySQL CONVERT_TZ() which fails on hosts without timezone tables installed (cPanel may or may not). The date list_filter still provides date-range filtering without the dependency.
Override Unfold template instead of monkey-patching Cleaner separation; survives Unfold upgrades as long as the upstream template path stays the same. If Unfold restructures, the override stops applying and we fall back to upstream behaviour (badge shows "None" again — visible but not fatal).
